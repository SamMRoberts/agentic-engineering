# agentic-engineering

A personal collection of agent customizations: plugins, instructions, custom agents, and the marketplace that distributes them. Built around the idea that agent quality comes from explicit workflows, deterministic validation, and small composable primitives — not large prompts.

## Repository layout

```
agentic-engineering/
├── plugins/                  # Installable plugins (this is what the marketplace serves)
│   ├── marketplace.json      # Auto-generated marketplace descriptor
│   ├── web-ux-testing/       # Web UX testing pack
│   └── web-frontend-testing/ # Playwright MCP frontend testing pack (incl. MCP App)
├── agents/                   # Standalone custom agents (not packaged as plugins)
├── instructions/             # Repository-scoped AGENTS.md instructions
│   ├── general-coding/       # Always-on coding discipline
│   └── agentic-tooling/      # Rules for editing agent tooling itself
├── scripts/
│   └── sync-marketplace.mjs  # Regenerates plugins/marketplace.json
├── .githooks/                # Repo-managed git hooks (opt-in per clone)
├── .github/
│   └── workflows/            # CI: validate plugins, sync marketplace
└── AGENTS.md                 # Workspace-wide agent guidance
```

The two most active areas are `plugins/` (full installable packs) and `instructions/` (repo-scoped guidance that applies to every change made in this workspace).

## Plugins

Each subfolder under `plugins/` is a self-contained pack that can be installed independently. Conventions are documented in [instructions/agentic-tooling/repo/AGENTS.md](instructions/agentic-tooling/repo/AGENTS.md), but the highlights:

- One user-facing orchestrator agent per plugin, with private sub-agents for stage work.
- Skills for repeatable steps, including bundled scripts/templates and deterministic validators.
- JSON Schemas + Node validators for any structured artifact a plugin produces.
- Lifecycle hooks for guarantees that instructions alone cannot enforce (credentials, plan validation, dangerous tool calls).
- Optional MCP App for interactive UI.

Current plugins:

| Plugin | Purpose |
| --- | --- |
| [web-ux-testing](plugins/web-ux-testing/) | Plan, run, and report structured web UX tests with Playwright MCP and CLI runners. |
| [web-frontend-testing](plugins/web-frontend-testing/) | Five-stage Playwright frontend testing workflow (intake → scan → plan → execute → report), plus an MCP App for triage and plan editing. |

## Using the marketplace

The marketplace descriptor lives at [plugins/marketplace.json](plugins/marketplace.json) and lists every plugin in `plugins/` with a `git-subdir` source pointing back at this repository. Any Claude Code (or compatible) marketplace reader can install from it.

### Add this marketplace locally

Add the entry below to your marketplace file (typically `~/.agents/plugins/marketplace.json` for Claude Code):

```jsonc
{
  "source": "https-fetch",
  "url": "https://raw.githubusercontent.com/SamMRoberts/agentic-engineering/main/plugins/marketplace.json"
}
```

…or clone this repo and point the host at the file path directly. Then install any listed plugin from the host's marketplace UI.

### Per-plugin overrides

The descriptor is generated, but each plugin can pin custom values by adding a `marketplace` block to its `.codex-plugin/plugin.json` (or root `plugin.json`):

```jsonc
{
  "name": "my-plugin",
  "marketplace": {
    "ref": "v0.2.0",
    "policy": { "installation": "PREVIEW", "authentication": "NONE" },
    "category": "Quality"
  }
}
```

Defaults are `ref: main`, `policy: AVAILABLE / ON_INSTALL`, and category inferred from `interface.category`.

## Marketplace sync

`plugins/marketplace.json` is generated by [scripts/sync-marketplace.mjs](scripts/sync-marketplace.mjs). Three layers keep it in step with `plugins/`:

1. **Local pre-commit hook** — [.githooks/pre-commit](.githooks/pre-commit) runs `--check` when staged files touch `plugins/` or the script. Opt in once per clone:
   ```bash
   git config core.hooksPath .githooks
   ```
2. **PR check** — [.github/workflows/sync-marketplace.yml](.github/workflows/sync-marketplace.yml) runs `node scripts/sync-marketplace.mjs --check` on every pull request that touches `plugins/` or the script. Configure it as a required status check on `main` so stale PRs cannot merge.
3. **Auto-commit on `main`** — the same workflow's `sync` job runs on push to `main`, regenerates the file, and commits any drift back with `[skip ci]`. This catches direct pushes and ensures the descriptor is always current within seconds.

Manual commands:

```bash
# Regenerate plugins/marketplace.json
node scripts/sync-marketplace.mjs

# Fail (exit 1) if the file is stale — used by CI and the pre-commit hook
node scripts/sync-marketplace.mjs --check

# Write somewhere else (e.g. directly into Claude Code's local marketplace)
node scripts/sync-marketplace.mjs --out ~/.agents/plugins/marketplace.json
```

The script has zero npm dependencies — Node 20+ stdlib only.

## Contributing

Before opening a PR:

1. Read [AGENTS.md](AGENTS.md) and the relevant folder-scoped instructions:
   - [instructions/general-coding/repo/AGENTS.md](instructions/general-coding/repo/AGENTS.md) for general engineering rules.
   - [instructions/agentic-tooling/repo/AGENTS.md](instructions/agentic-tooling/repo/AGENTS.md) when changing any agent customization (plugins, skills, agents, hooks, schemas).
2. Enable the repo's git hooks once per clone:
   ```bash
   git config core.hooksPath .githooks
   ```
3. Make the smallest focused change that satisfies the requirement.
4. Run the relevant validators:
   - Plugin tests: `cd plugins/<plugin> && npm install && npm test`
   - Marketplace check: `node scripts/sync-marketplace.mjs --check`
5. Write a short PR description that names what changed and what validation you ran.

### Adding a new plugin

1. Create `plugins/<plugin-name>/` following the layout in [instructions/agentic-tooling/repo/AGENTS.md](instructions/agentic-tooling/repo/AGENTS.md).
2. Include at minimum `plugin.json`, `.codex-plugin/plugin.json` (rich metadata), and a user-facing entrypoint agent.
3. Regenerate the marketplace and commit the result:
   ```bash
   node scripts/sync-marketplace.mjs
   git add plugins/marketplace.json
   ```
4. Open a PR. CI will run the marketplace `--check` and any plugin-specific tests; the auto-commit job will re-sync if needed after merge.

### Modifying an existing plugin

- Bump the plugin's version in `package.json` and (where present) `.codex-plugin/plugin.json` following semver.
- Update `CHANGELOG.md` if the plugin has one.
- Run the plugin's own tests and validators.
- The marketplace is regenerated automatically; no manual edit needed unless you're tweaking a `marketplace` override block.
