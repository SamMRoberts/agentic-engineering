# Web UX Testing Best Practices

- Prefer observable user-facing signals.
- Use roles, labels, text, and `data-testid` locators.
- Include authentication state handling.
- Include loading, empty, error, and success states.
- Capture console errors and failed network requests.
- Include keyboard-only navigation.
- Include accessible names and focus management.
- Test reload and browser history behavior.
- Avoid destructive production actions.
- Convert stable high-value scenarios into Playwright CLI tests.
